### #include三种写法

- `#include <iostream.h>`（现代是 `<iostream>`）

​	尖括号：走**系统头文件路径**（编译器内建 include 目录）

- `#include "Complex.h"`

 	双引号：优先**当前项目目录/当前文件目录**，再走系统路径

- `#include "d:\课堂例程\Complex.h"`

​	绝对路径指定：不推荐（可移植性差，换电脑/换目录就炸）



### 条件编译



语法:

```c++
#if 表达式
#elif 表达式
#else
#endif
```



**表达式的语法**

1. **#if 和 #elif 接的表达式**中可以含有如 **defined 标识符** 或 **defined(标识符)** 的一元运算符，当此标识符已经被定义为宏名时，结果为 1，否则结果为 0。当整个表达式求值为非0时，编译其所控制的代码块，否则不编译该代码块。

```c++
#define a ll
#include<iostream>
using namespace std;
signed main() {
#if defined a 
	cout << 1 << endl;
#endif
}
```

2. #if 和 #elif 接的表达式中也可以含有__has_include(C++17 起) 或 __has_cpp_attribute(C++20 起) 表达式。

   __has_include 表达式，检测某个头文件或源文件是否存在

   __has_cpp_attribute 表达式，检测给定属性记号是否受支持和支持的版本。

```c++
#if __has_include(<iostream>) 
	cout << 1 << endl;
#endif
```



### #pragma once

假如A.h被B.h和C.h包含 如果main.cpp被B.h和C.h都包含了就会报重定义错误

加了pragma once就相当于这个头文件只会被包含一次

等价于

```c++
#ifndef A_H
#define A_H
加入头文件
#endif
```



### 类名应用场景及其限制

如果只有类的声明,**只能知道类的名字不能知道类的成员**

```c++
class A;
void func(A*p);//只能得到类指针和引用
class B{
    friend class A;//以及声明友元类
};
```

**必须有类的定义/include"A.h"才能访问类对象和成员**
